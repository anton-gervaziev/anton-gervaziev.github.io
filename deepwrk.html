Why It Was Rebuilt

The original Bubble.io implementation made early development fast, but by the time the platform had real users, two problems were getting worse.

Performance: Bubble's runtime adds abstraction on top of every database call. Page loads were hitting 3–8 seconds. Real-time features — live session counts, chat — had 2–4 second delays. For a product whose entire value proposition is getting into flow state, that's fatal.

Maintainability: Every new feature required working around Bubble's constraints. What should have been a two-hour feature took two days. The codebase was a maze of workarounds with no exit.

Technical Challenges
01

Data Migration from Bubble

Bubble stores data in its own format — no SQL export. I wrote scripts to pull everything out, reshape it, and load it into the new schema. Nothing lost.

02

Zero-Downtime Cutover

200+ paying subscribers on the old system. Ran both in parallel, checked the data matched, then switched over. Nobody lost anything.

03

Reproducing Implicit Logic

Bubble's visual workflows encode logic that's hard to read and impossible to export. Reverse-engineered behavior by testing the running app, re-implemented in C#.

04

Auth Rebuild

Bubble's auth is a black box. Replaced it with ASP.NET Identity — migrated all existing accounts, handled credential resets, added role-based access.
